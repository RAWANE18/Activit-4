    les techniques de l’information et de la communication :

    les techniques de l’information et de la communication (TIC) comme l'ensemble des outils matériels et logiciels permettant de manipuler, produire, transmettre, échanger, stocker, et diffuser des informations sous forme vocale, images ou données contenues dans des signaux de nature optiques, électromagnétiques ou acoustiques. Les TIC ont vu le jour grâce à la convergence de l'informatique, des télécommunications, de l'électronique et de l'audiovisuel.

    Catégories et objectifs :

    Les TIC se divisent en trois catégories principales : la téléinformatique, les télécommunications et l'électronique. Les objectifs de TIC comprennent la connectivité, la modularité, l'implantation simple, l'utilisation facile et la fiabilité.

    Composants de TIC :

    Les composants de TIC comprennent la partie matérielle (ordinateurs, télévisions, téléphones portables, tablettes, appareils photo numériques, robots, etc.) et la partie logicielle (systèmes d'exploitation, logiciels d'application et d'utilisation des différents composants matériels, réseaux internet ou mobiles).

    Caractéristiques :

    Les caractéristiques de TIC comprennent la rapidité, la décentralisation du pouvoir, un meilleur accès aux informations, la rationalité, la mondialisation et la répartition de l'information.

    Initiation aux techniques de communication :

    - Les objectifs de communication comprennent la prise de conscience, la modification de l'image et le changement de comportement.
    - Le système de communication comprend les partenaires, les vecteurs, les fonctions et le bruit.
    - Il y a deux types de communication : institutionnelle et commerciale.
    - Il y a également trois schémas de communication : interpersonnel, de groupe et de masse .

    Les cartes réseau :

    des composants installés dans les ordinateurs pour permettre la communication avec un réseau.

    Les switches :

    des composants utilisés pour connecter des appareils sur un réseau local et permettre la communication entre eux.

    Le modèle OSI :

    un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes.

    Le modèle TCP/IP :

    un modèle de référence pour les communications de réseau qui divise les communications en quatre couches principales.

    Les adresses IP :

    des numéros qui identifient chaque ordinateur sur un réseau.

    Le masque de réseau et de sous-réseaux :

    une valeur utilisée pour identifier le réseau et les sous-réseaux d'une adresse IP. En utilisant ces différents composants et modèles, les ordinateurs peuvent communiquer et échanger des données sur un réseau.

    Internet et Web

    - Le World Wide Web, communément appelé le Web, est un système hypertexte public sur Internet qui permet de consulter des pages accessibles sur des sites.
    -Le Web a été inventé par Tim Berners-Lee et Robert Cailliau plusieurs années après la création d'Internet.
    - Les pages Web sont décrites élément par élément à l'aide de langages de description tels que HTML, XHTML ou HTML5, et sont stockées dans des fichiers HTML.
    -Ces pages sont interprétées et affichées par des navigateurs, et sont accessibles à partir d'URL qui contiennent l'adresse du serveur et l'adresse de la page sur le serveur.
    - Un site web est un regroupement de pages Web autour d'une thématique commune, reliées entre elles par des liens hypertextes, et est généralement localisé sur un même serveur.
    - La création d'une page web HTML implique la construction d'un fichier texte suivant un format spécifique, et peut inclure du texte, des images, des liens hypertextes, et même du contenu interactif tel que des applications ou des discussions en ligne.

    INTRODUCTION :

    Ce document a pour but de clarifier les concepts de Git et GitHub pour éviter toute confusion. Nous explorerons également la popularité de Git et la nécessité des systèmes de gestion de version dans le monde actuel.
    Qu’est-ce que Git ?
    Git est un logiciel de versioning créé en 2005 par Linus Torvalds le createur de LINUX.
    Un logiciel de version permet de suivre les modifications effectuées sur un projet, ce qui est essentiel pour la gestion de projets et le travail en équipe. Git est actuellement le logiciel de gestion de version le plus populaire et représente un outil indispensable pour tout développeur.

    A quoi sert concrètement un système de gestion de version ?

    Pour illustrer l’utilité d’un logiciel de gestion de version, prenons l'exemple d'un site web que vous souhaitez modifier ou tester une nouvelle fonctionnalité. Sans un système de gestion de version, vous devrez effectuer une sauvegarde complète du site avant chaque modification pour pouvoir le restaurer en cas de problème. Cela devient encore plus compliqué lorsque plusieurs personnes travaillent simultanément sur le même site web en développant des fonctionnalités différentes. Un système de gestion de version est une solution pour gérer efficacement les modifications apportées au code source en conservant un historique de ces modifications. Git est un logiciel de gestion de version largement utilisé qui permet aux développeurs de travailler simultanément sur un même projet tout en gardant une trace de toutes les modifications apportées aux fichiers, et permettant de revenir à une version antérieure si nécessaire. Avec Git, les développeurs peuvent collaborer de manière plus efficace et productive.

    Les deux modèles des logiciels de gestion de version :

    modèle centralisé vs modèle décentralisé
    Les logiciels de gestion de version sont aujourd’hui tous construits sur l’un des deux modèles suivants : le modèle centralisé ou le modèle décentralisé encore appelé modèle distribué. Dans le modèle centralisé, le code source est hébergé sur un serveur distant central et chaque utilisateur doit se connecter à ce serveur pour travailler sur le code. En revanche, dans le modèle décentralisé, chaque utilisateur télécharge et héberge le code source sur sa propre machine, même si le code source reste hébergé sur un serveur distant. Le modèle décentralisé offre des avantages notables tels que la simplicité et la flexibilité du travail ainsi que la sécurité accrue grâce aux sauvegardes des copies du projet de chaque utilisateur. Git est un exemple de logiciel de gestion de version basé sur le modèle décentralisé.

    Qu’est-ce que GitHub ?

    Git est un outil de gestion de version permettant d'enregistrer les modifications d'un projet et de revenir à une version précédente. On appelle un dépôt la copie des fichiers du projet et de leur version stockée sur un serveur central, Git appelle ça également “repository” ou “repo” en abrégé. GitHub est le plus grand hébergeur de dépôts Git, où la plupart sont publics et ouverts à la contribution. En bref, Git gère les versions et GitHub héberge les dépôts.

    Utiliser Git : ligne de commande, console et interface graphique :

    Le cours utilise la console en ligne de commande pour installer et utiliser Git plutôt qu'une interface graphique car cela donne accès à toutes les commandes Git, assure une cohérence dans l'apprentissage et permet aux utilisateurs de se familiariser avec la ligne de commande, ce qui facilite l'utilisation de toutes les autres interfaces graphiques.
    En Windows, pour ouvrir la ligne de commande :
    1-Raccourcie : [TOUCHE WINDOWS]+R+ cmd
    2-Dans la barre de recherche, rechercher cmd

    Installation de Git :

    Téléchargez et installez la dernière version de Git sur http://git-scm.com/downloads en acceptant les valeurs par défaut.

    Paramétrage de Git :

    Après l'installation de Git, configurez votre nom d'utilisateur et adresse mail en utilisant la commande git config. La commande git config permet de voir et modifier les variables de configuration de Git. Utilisez l'option --global pour appliquer ces paramètres à tous vos projets Git. Les commandes à taper sont : git config --global [user.name](http://user.name/) "Votre Nom" et git config --global user.email [votre@email.com](mailto:votre@email.com). Remplacez "Votre Nom" et "[votre@email.com](mailto:votre@email.com)" par vos propres informations.
    Pour vous assurer que vos informations ont bien été enregistrées, vous pouvez taper git config [user.name](http://user.name/) et git config user.email. Les informations entrées devraient être renvoyées.

    Démarrer un dépôt Git :

    Git permet de copier et importer tous les fichiers d'un projet dans un "dépôt". Il y a deux façons de créer un dépôt : importer un répertoire existant ou cloner un dépôt existant. Le cours expliquera comment faire cela, mais avant cela, il est important de comprendre le fonctionnement général de Git.

    La gestion des informations selon Git :

    Git considère les données sous forme d'instantanés ou de "snapshots". À chaque validation, Git prend une capture du contenu de l'espace de travail et enregistre une référence à cet instantané. Chaque instantané est stocké localement dans une base de données. Avoir l'historique complet d'un projet localement permet de réaliser la plupart des opérations de Git localement, sans avoir besoin d'être connecté à un serveur central distant. Cela rend les opérations plus rapides et le travail plus agréable.

    Les états des fichiers :

    Git suit les fichiers d'un projet en les classant en deux états : "suivi" ou "non suivi". Un fichier est "suivi" s'il appartient au dernier instantané enregistré, sinon il est "non suivi". Les fichiers sont "non suivis" au démarrage d'un nouveau dépôt, mais "suivis" lorsqu'on clone un dépôt existant. Les fichiers "suivis" peuvent être modifiés, indexés ou validés.
    chaque fichier suivi peut avoir l’un de ces trois états :
  • Modifié (“modified”);
  • Indexé (“staged”);
  • Validé (“committed”).
    • Lorsqu'on utilise Git pour gérer un projet, on commence par valider tous les fichiers du projet en les stockant localement. Lorsqu'on travaille sur le projet, les fichiers modifiés sont considérés comme tels par Git jusqu'à ce qu'ils soient indexés. Pour indexer un fichier, on lui indique qu'il doit faire partie du prochain instantané dans sa version actuelle. Lorsqu'on prend un instantané de l'état actuel du projet, les fichiers indexés et non modifiés sont considérés comme validés, et le processus recommence. Lorsqu'on clone un dépôt existant, les fichiers sont automatiquement enregistrés et validés.

    Les zones de travail :

    Git est un système de gestion de versions de projet qui utilise des états de fichiers pour suivre les changements de code et de fichiers au fil du temps. Le projet est composé de trois parties principales: le répertoire de travail, la zone d'index et le répertoire Git.
    Le répertoire de travail est une version extraite du projet à un instant précis. La zone d'index est un fichier dans le répertoire Git qui stocke les informations sur les fichiers qui seront inclus dans le prochain commit. Le répertoire Git est l'endroit où Git stocke les méta-données et la base de données des objets du projet.
    Pour travailler avec Git, on modifie ou crée des fichiers dans le répertoire de travail, puis on les indexe pour leur donner l'état "indexé". Les fichiers non indexés sont soit dans l'état "modifié" s'ils ont été modifiés depuis le dernier commit, soit dans l'état "non suivi" s'ils sont nouveaux. Enfin, on valide la version indexée de nos fichiers en effectuant un "commit" pour les ajouter au répertoire Git.

    Créer un dépôt Git à partir d’un répertoire existant :

    En démarrant avec Git, on peut déjà avoir des projets en cours stockés localement ou sur un serveur distant. Il est possible d'implémenter un système de gestion de version en important l'ensemble des ressources du projet dans Git. Pour cela, je vais créer un répertoire nommé "projet-git" sur mon bureau, contenant deux fichiers texte vides nommés "fichier1.txt" et "README.txt".
    Ensuite, pour initialiser un dépôt Git, il suffit d'utiliser la commande "git init", qui va créer un sous-répertoire ".git" contenant les fichiers nécessaires pour le fonctionnement du dépôt Git.
    En cmd, on peut utiliser les commandes suivantes :
    - "cd path" pour se déplacer dans un répertoire
    - "dir" pour afficher la liste des dossiers et fichiers dans l'emplacement actuel.
    Lorsqu'on utilie "git init", Git nous informe que le dépôt Git a été initialisé et qu'il est vide. C'est normal car aucun fichier n'a encore été versionné (ni ajouté à la base). Pour déterminer l'état des fichiers du répertoire, on peut utiliser la commande "git status", qui est très utile.

    Cloner un dépôt Git

    Pour cloner un dépôt Git, suivez ces étapes :
    Ouvrez votre terminal (ou ligne de commande) et placez-vous dans le répertoire où vous souhaitez cloner le dépôt.
    Utilisez la commande git clone suivie de l'URL du dépôt que vous souhaitez cloner. Par exemple :
    git clone https://github.com/nom_utilisateur/nom_du_depot.git
    Cette commande va télécharger tous les fichiers et l'historique du dépôt dans un nouveau dossier portant le nom du dépôt.
    Si le dépôt est privé, Git vous demandera de fournir vos identifiants GitHub (ou autres) pour vous authentifier.

    Ajouter ou modifier des fichiers dans un projet et actualise

    Nous avons un projet contenant des fichiers et un dépôt Git qui permet de gérer les versions du projet. En travaillant sur le projet, nous ajouterons, modifierons ou supprimerons des fichiers. Pour conserver un historique des versions, nous utilisons les commandes "git add" et "git commit". Il est important de noter que le commit d'un fichier se base sur son état au moment du "git add". Si nous voulons enregistrer la dernière version d'un fichier, nous devons effectuer un "git add" juste avant le "git commit". Pour simplifier le processus, nous pouvons utiliser "git commit -a" pour ajouter les fichiers modifiés et déjà suivis dans la zone d'index et les valider en un seul coup.

    Consulter l’historique des modifications Git :

    La commande git log permet de consulter l'historique des modifications de manière simple. Elle affiche la liste des commits réalisés du plus récent au plus ancien avec des informations comme la somme de contrôle SHA-1, le nom et l'e-mail de l'auteur, la date et le message du commit.

    Annuler des modifications apportées à un fichier :

    L'un des avantages de la gestion de version est de pouvoir revenir à un état précédent d'un projet. Après un commit, si des modifications ne sont pas satisfaisantes, on peut utiliser la commande git checkout -- nom-du-fichier ou la commande spécialisée git restore pour revenir à l'état précédent du fichier.

    Qu’est ce qu’une branche ?

    Une branche est simplement un pointeur vers un commit, qui est un objet stockant l'instantané du contenu indexé et des pointeurs vers les commits précédents. La branche par défaut dans Git s'appelle "master", mais on peut facilement créer de nouvelles branches pour développer et tester de nouvelles fonctionnalités sans impacter le projet principal. La création d'une branche crée simplement un nouveau pointeur vers un commit spécifique, ce qui rend la création et la fusion de branches très facile dans Git.
